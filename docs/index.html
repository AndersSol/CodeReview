CodeReview

Multi-AI security review framework — evidence-first pipeline with deterministic consolidation, severity rubric, and release-policy engine.

Run three AI agents (Codex, Claude, optional Bob) through structured security review of your codebase. Get an HTML report with executive summary, full audit log, and machine-readable consolidation — without trusting any single agent.

Install

One command, requires Python 3.13+ and pipx:

curl -fsSL https://anderssol.github.io/CodeReview/install.sh | bash

The install script verifies SHA-256 against latest.json before installing.

What you get

  • atea-security-review — pipeline CLI for consolidating agent outputs into HTML report + JSON audit log
  • atea-preflight — deterministic preflight (sandbox + inventory + optional gitleaks/bandit/semgrep/pip-audit + prompt-injection scan)
  • Claude Code skill at ~/.claude/skills/security-review (invoke with /security-review <path>)
  • Bundled v2 JSON schema + v5 HTML template + 8-rule default release policy

How it works

  1. Preflight: sandbox-validate target, inventory files, run optional scanners (gitleaks/bandit/semgrep/pip-audit), scan for prompt-injection in source comments
  2. Parallel agent run: Codex (code reasoner) + Claude (threat-model reviewer) + Bob (adversarial reviewer, optional) get the same hashed prompt with preflight context
  3. Schema validation: each agent output validated against v2 schema; malformed outputs excluded gracefully
  4. Per-finding evidence verification BEFORE consolidation: open source file, confirm quote at claimed line
  5. Deterministic consolidation: cluster + dedup + worst-tuple rubric merge
  6. Severity from rubric: impact_class × privilege × env × input × trigger → severity. Not from agent vote.
  7. Confidence model: evidence_grade (55%) + reachability (25%) + agreement (15%) + tool_origin (5%). No agent self-confidence.
  8. Assessment classifier: 5 states (confirmed / plausible / weak_evidence / needs_manual_review / false_positive_candidate)
  9. Release policy engine: 8 default rules, all evaluated, first-match wins. Override via --release-policy <file.json>.
  10. HTML report with per-finding business-risk narrative + audit log JSON with all hashes for reproducibility

Requirements

  • Python 3.13+
  • pipx (brew install pipx)
  • Codex CLI (required) — primary code-reasoner agent
  • Claude Code (required) — threat-model reviewer (sub-agent)
  • Bob / IBM Bob Shell (optional) — adversarial reviewer
  • Optional scanners: gitleaks, bandit, semgrep, pip-audit

Verifying a release

curl -fsSL https://anderssol.github.io/CodeReview/latest.json
# version, wheel_url, sha256, size_bytes, python_requires, released

Install script downloads from wheel_url, recomputes SHA-256, fails hard on mismatch.
